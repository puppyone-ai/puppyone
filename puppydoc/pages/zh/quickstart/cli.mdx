# CLI 快速开始

5 分钟用命令行跑通完整工作流。

---

## 安装

```bash
npm install -g puppyone
```

验证安装：

```bash
puppyone --version
```

---

## 登录

```bash
puppyone auth login -e user@example.com -p password
```

登录成功后，CLI 会自动保存凭证，后续命令无需重复认证。

---

## 设置组织和项目

```bash
puppyone org use "My Organization"
puppyone project use "My Project"
```

设置完成后，后续所有操作都会默认作用于当前项目。

---

## 浏览和创建内容

PuppyOne 中的数据以 Content Node 树形式组织，支持文件夹、JSON、Markdown 和文件四种类型。内容操作通过 REST API 或 Dashboard 完成：

```bash
BASE_URL="https://api.puppyone.ai/api/v1/content/{project_id}"

# 列出根目录
curl "$BASE_URL/ls" -H "Authorization: Bearer $TOKEN"

# 创建文件夹
curl -X POST "$BASE_URL/mkdir" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"path": "docs"}'

# 写入 Markdown 文件
curl -X POST "$BASE_URL/write" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"path": "docs/readme.md", "content": "# Hello World", "node_type": "markdown"}'

# 写入 JSON 文件
curl -X POST "$BASE_URL/write" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"path": "products/list.json", "content": [{"name": "Widget", "price": 99}], "node_type": "json"}'
```

也可以在 Dashboard 编辑器中直接创建和编辑内容。

## 连接数据源

从外部平台同步数据到你的 Context File System。

```bash
# 连接 Notion 数据源（会自动引导完成 OAuth 授权）
puppyone access add notion https://notion.so/your-database-id --folder /docs
```

> **Note:** Notion OAuth 授权通过 Dashboard 或 `puppyone access add notion` 流程自动完成，无需单独执行授权命令。

支持 Notion、GitHub、Gmail、Google Drive、Linear、Airtable 等 15+ 平台。

---

## 创建 MCP 端点

为 Cursor、Claude Desktop 等 MCP 客户端创建接入点。

```bash
puppyone access add mcp "My API"
```

创建后会返回 MCP Server URL 和 API Key，将其配置到客户端即可。

## 同步本地文件夹

通过 MUT 协议将本地文件夹同步到云端，支持版本控制。

```bash
# 已经有想同步的本地文件夹？一步到位：
puppyone access add filesystem docs --link ~/my-docs

# 或者先建 AP，再二选一：clone 全新 / connect 已有
puppyone access add filesystem docs
# 输出会同时给出两条路径，挑你的场景：
mut clone <url> --credential <key>                       # A) 新建文件夹
cd ~/my-docs && mut connect <url> --credential <key>     # B) 已有文件夹

# 然后编辑、提交、推送
cd ~/my-docs
echo "# Hello" > readme.md
mut commit -m "init"
mut push
```

### 安装 MUT CLI

最常见的方式：

```bash
pip install mutai
```

#### Ubuntu 24.04 / Debian 12+ —「找不到 pip」或「externally-managed-environment」

新版 Ubuntu/Debian 默认没装 `pip`，并且按 PEP 668 规则禁止用系统 pip 全局装包。任选其一：

```bash
# 推荐：用 pipx 装 CLI 工具，自动隔离环境
sudo apt update && sudo apt install -y pipx
pipx ensurepath
pipx install mut

# 或者：装 pip 然后用 venv 隔离
sudo apt install -y python3-pip python3-venv
python3 -m venv ~/.venvs/mut
source ~/.venvs/mut/bin/activate
pip install mutai

# 不推荐：强行突破 PEP 668（会污染系统 Python）
pip install --break-system-packages mut
```

> 装完用 `mut --version` 验证。如果 `which mut` 找不到，把 `~/.local/bin` 加进 PATH：`export PATH="$HOME/.local/bin:$PATH"`。

### 大文件 / 慢网络：调高 MUT_TIMEOUT

`mut push` 默认 socket 超时是 60 秒。如果是大 commit（比如几十 MB 的 PDF / 视频）或者慢网络，会被吊死并报「cannot reach server」。把超时调大：

```bash
# 单次 push（5 分钟）
MUT_TIMEOUT=300 mut push

# 当前 shell 都生效
export MUT_TIMEOUT=300
```

> 服务器单次 push 上限是 200 MB（单文件 50 MB）。超过会返回 413，附带怎么拆 commit 的提示。

## 完整示例

以下是从零开始的完整操作流程：

```bash
# 安装并登录
npm install -g puppyone
puppyone auth login -e user@example.com -p password

# 选择项目
puppyone org use "My Organization"
puppyone project use "My Project"

# 通过 REST API 创建目录结构和写入内容
BASE_URL="https://api.puppyone.ai/api/v1/content/{project_id}"

curl -X POST "$BASE_URL/mkdir" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"path": "docs"}'

curl -X POST "$BASE_URL/write" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"path": "docs/readme.md", "content": "# 产品知识库", "node_type": "markdown"}'

curl -X POST "$BASE_URL/write" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"path": "products/catalog.json", "content": [{"name": "Widget Pro", "price": 99.99}], "node_type": "json"}'

# 连接 Notion 数据（OAuth 授权会自动引导完成）
puppyone access add notion https://notion.so/db-id --folder /docs/notion

# 为 Agent 创建 MCP 端点
puppyone access add mcp "Production API"

# 同步本地文件夹（一步到位 —— 把已有文件夹接入云端）
puppyone access add filesystem workspace --link ~/workspace
# 如果本地还没有该文件夹，跟着输出里的 `mut clone` 命令跑

# 检查结果
puppyone access ls
puppyone status
```
