# 版本控制

像 Git 一样管理你的 Context File System，每一次变更都自动留痕。

---

## 为什么需要版本控制

当多个 Agent 同时操作你的数据时，意外覆盖、误删、写入错误内容都可能发生。PuppyOne 的版本控制系统为每一次变更自动创建版本化的 commit，让你可以：

- **随时回退** — 通过 API 恢复到任意历史版本
- **审查变更** — 对比两个版本的差异，确认 Agent 做了什么
- **全程追溯** — 所有操作记录在审计日志中

不需要手动提交、不需要记住保存——所有变更自动记录。

---

## 核心能力

### Mut Commits

每次通过 Content API 的写入都会创建一个版本化的 commit，存储在 `mut_commits` 表中。每个 commit 记录了变更内容、操作者、时间戳和可选的提交信息。

```bash
# 查看文件的 commit 历史
curl -X GET "https://api.puppyone.ai/api/v1/content/{project_id}/versions?path=/docs/product-spec.md" \
  -H "Authorization: Bearer {token}"
```

### 版本对比

将任意两个版本并排对比，清晰看到新增、删除和修改的内容。

```bash
# 对比版本 2 和版本 5
curl -X GET "https://api.puppyone.ai/api/v1/content/{project_id}/diff?path=/docs/readme.md&v1=2&v2=5" \
  -H "Authorization: Bearer {token}"
```

### 回滚

将文件恢复到任意历史版本。回滚操作本身也会创建一个新的 commit，历史永不丢失。

```bash
# 回滚到版本 3
curl -X POST "https://api.puppyone.ai/api/v1/content/{project_id}/rollback" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{"path": "/docs/product-spec.md", "version": 3}'
```

### 审计日志

所有操作都记录在审计日志中，包括操作者、变更内容和时间。

---

## 适用场景

| 场景 | 推荐功能 |
|------|----------|
| Agent 意外覆盖了文件 | 回滚到之前的版本 |
| 审查 Agent 的修改内容 | 版本对比 |
| 追踪数据随时间的演变 | 版本历史（commits） |
| 排查数据异常 | 审计日志 + 版本对比 |

---

## 与 Git 的区别

PuppyOne 的版本控制专为**云端 Context File System** 设计，和 Git 有几个关键差异：

- **自动版本化** — 每次写入自动创建 commit，无需 `commit` 命令
- **基于路径寻址** — 文件通过 project ID 和路径标识，而非 UUID
- **无分支概念** — 线性版本历史，简化 Agent 的使用场景
- **即时回滚** — 回滚操作本身也会创建新 commit，历史永不丢失
