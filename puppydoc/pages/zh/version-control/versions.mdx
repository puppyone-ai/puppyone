# 文件版本历史与回滚

每次写入自动创建版本化的 commit，随时查看历史、恢复到任意时间点。

---

## 自动版本化

每次对文件（JSON、Markdown 或其他内容）的修改通过 Content API 完成时，PuppyOne 都会自动：

1. 为当前内容创建一个新 commit
2. 记录修改时间和操作者（人类用户或 Agent）
3. 保留完整的历史 commit 链

不需要手动保存或提交。无论是通过 Dashboard 编辑、API 调用，还是 Agent 通过 MCP 写入，所有变更都会被追踪。

---

## 查看版本历史

### Dashboard

1. 打开 Project → 选择一个文件
2. 点击右侧面板的 **History** 标签
3. 查看版本列表，包含版本号、时间和操作者

### API

```bash
# 获取文件的版本历史
curl -X GET "https://api.puppyone.ai/api/v1/content/{project_id}/versions?path=/docs/product-spec.md" \
  -H "Authorization: Bearer {token}"
```

响应示例：

```json
{
  "versions": [
    {
      "version": 5,
      "created_at": "2025-03-09T15:30:00Z",
      "author": "Agent: data-sync",
      "size": 2048
    },
    {
      "version": 4,
      "created_at": "2025-03-09T14:00:00Z",
      "author": "user@example.com",
      "size": 1856
    }
  ]
}
```

## 查看特定版本内容

### API

```bash
# 获取版本 3 的完整内容
curl -X GET "https://api.puppyone.ai/api/v1/content/{project_id}/versions?path=/docs/product-spec.md&version=3" \
  -H "Authorization: Bearer {token}"
```

响应会返回该版本的完整文件内容，可以用来预览或手动恢复部分数据。

---

## 回滚到历史版本

回滚会将文件恢复到指定版本的内容。**回滚操作本身也会创建一个新版本**，所以历史记录永远不会丢失。

### Dashboard

1. 打开文件的 **History** 面板
2. 选择要恢复的版本
3. 点击 **Restore**
4. 确认回滚

### API

```bash
# 回滚到版本 3
curl -X POST "https://api.puppyone.ai/api/v1/content/{project_id}/rollback" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{"path": "/docs/product-spec.md", "version": 3}'
```

## 实际场景：Agent 误操作恢复

假设你让一个 Agent 更新产品文档，但它意外覆盖了整个文件内容：

```bash
# 1. 查看版本历史，确认问题
curl -X GET "https://api.puppyone.ai/api/v1/content/{project_id}/versions?path=/docs/product-spec.md" \
  -H "Authorization: Bearer {token}"

# 输出显示 v8 是 Agent 的错误写入
# v7 是正确的版本

# 2. 对比版本，确认差异
curl -X GET "https://api.puppyone.ai/api/v1/content/{project_id}/diff?path=/docs/product-spec.md&v1=7&v2=8" \
  -H "Authorization: Bearer {token}"

# 3. 回滚到正确版本
curl -X POST "https://api.puppyone.ai/api/v1/content/{project_id}/rollback" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{"path": "/docs/product-spec.md", "version": 7}'
```

整个恢复过程不到一分钟，数据零丢失。

---

## 注意事项

- 回滚不会删除任何历史版本，它创建一个新版本，内容与目标版本相同
- 版本号是递增的，回滚后的新版本号会继续递增
- 每个 Content Node 独立管理版本，修改一个文件不会影响其他文件的版本历史
- 版本历史记录操作者信息，方便在[审计日志](/zh/audit)中追溯
