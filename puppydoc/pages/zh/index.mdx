# PuppyOne

**为 AI Agent 打造的 Context File System。**

连接、管控、分发——让所有 Agent 共享同一份上下文。

---

## Agent 需要的不只是文件系统

今天最强的 Agent 都以文件为核心：它们通过 Bash 和本地文件系统来读、写、执行。但**传统文件系统从来不是为 Agent 设计的上下文基础设施。**

- **没有 Connector** — 你的数据散落在 Notion、GitHub、Google Drive 和几十个工具里，Agent 看不到其中任何一个。
- **没有备份和回滚** — 一个幻觉中的 Agent 覆盖了关键文件，之前的版本就没了。
- **没有文件级权限** — 靠 `chmod` 和 SSH Key 控制每个 Agent 的读写范围？完全没法扩展。

---

## PuppyOne 怎么解决

PuppyOne 是一个只为 Agent 而生的 Context File System，直接解决上面所有问题：

- **15+ Connector** — 把 Notion、GitHub、Gmail、Google Drive、Airtable 等平台的数据挂载到一棵统一的文件树里，自动转换为 Agent 友好的格式（Markdown、JSON、原始文件）。
- **版本控制与回滚** — 每次写入自动留痕。对比任意历史版本的差异，一键回滚。
- **Auth for Agents** — 每个 Agent 基于自己的身份，看到属于自己的文件树视图。没有权限的路径，物理上不存在。
- **审计日志** — 完整可追溯：哪个 Agent 在什么时候读取、修改或删除了哪个文件。
- **多通道访问** — 通过 MCP、本地文件夹同步、REST API、CLI 或沙盒分发你的上下文。Agent 用最适合它的方式接入。
