## PuppyOne 能做什么

PuppyOne 将你组织内分散的数据统一为一个**单一的、结构化的上下文基座 (Context Base)**，并实时分发给任何 AI Agent。

### 1. 集中上下文 (The "One" Base)
与其让每个 Agent 单独连接每个数据源，不如将它们全部连接到 PuppyOne。
- **连接器**: Notion, GitHub, Airtable, Google Sheets, Linear, Slack。
- **文件**: PDF, DOCX, Markdown, 代码库。
- **结构化**: 自动 ETL 流水线将原始数据清洗并组织为结构化 JSON。

### 2. 全渠道分发 (The Interface)
我们为每种类型的 Agent 提供最合适的接口：

| 接口 | 适用场景 |
|-----|----------|
| **MCP Server** | **Cursor, Claude Desktop**。原生集成，零代码接入。 |
| **REST API** | **自定义 Agent (Python/Node)**。完整的编程访问能力。 |
| **Cloud Skills** | **ChatGPT, Coze**。为托管 Agent 提供的即插即用技能。 |
| **Sandbox** | **安全执行**。让 Agent 在安全环境中针对数据运行代码。 |

### 3. Agent 原生管控
- **确定性索引**: Agent 需要精确路径和结构化查询，而不仅仅是模糊的向量搜索。
- **权限范围**: 给客服 Agent `/docs` 的只读权限，但给开发 Agent `/code` 的读写权限。

---

## 为什么需要 PuppyOne

- **消除幻觉**: 在处理价格、规格和代码时，结构化上下文优于向量搜索。
- **统一 Agent 记忆**: 当开发 Agent 更新了需求文档，QA Agent 能瞬间看到。
- **安全与治理**: 在一个地方统一管理所有 Agent 的可见范围和操作权限。
