# puppyone

**AI Agent 的 Context File System。**

把分散在各处的数据，同步成 Agent 可以直接读写的结构化文件系统。

---

## 一切皆 Sync

puppyone 的核心只有一件事：**Synchronize**。

不管是数据从 Notion 同步进来，还是 Agent 通过 MCP 读写，还是本地文件夹通过 OpenClaw 双向同步——底层都是同一个同步引擎，只是方向和协议不同。

```
  Notion  GitHub  Gmail  Linear  Files
     │       │       │      │      │
     └───────┴───────┴──────┴──────┘
                     │
               ← Sync (Pull)
                     │
            ╔════════▼════════╗
            ║    puppyone     ║   Context File System
            ╚════════▲════════╝
                     │
               Sync (Read / Write) →
                     │
       ┌─────────────┼─────────────┐
       │             │             │
   OpenClaw         MCP         REST API
   (↕ 双向)      (读/写)        (读/写)
       │             │             │
  本地文件夹   Cursor / Claude   自研 Agent
```

MCP 和 REST API 不是独立的"分发层"——它们是 sync 结果之上的协议封装。OpenClaw 则是最直接的双向 sync，本地文件夹和 puppyone 实时保持一致。

---

## File Level Security (FLS)

**把权限控制精确到文件路径。**

类似数据库的行级权限（RLS），FLS 控制每个 Agent 能看到哪些路径、能做哪些操作。

- 客服 Agent 只能读 `/products` 和 `/faq`
- 研发 Agent 可以读写 `/specs`，但不能删除
- 销售 Agent 能看报价但看不到 `/internal`

Agent 没有权限的路径，**在它的视野中物理上不存在**。
