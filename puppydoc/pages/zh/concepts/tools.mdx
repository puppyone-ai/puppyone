# 工具与权限

控制你的 AI Agent 能对数据做什么。

## 工具类型

PuppyOne 提供细粒度的 Agent 能力控制：

| 工具 | 类型 | 描述 |
|-----|------|-----|
| `query_data` | 读 | 按 JSON 路径过滤查询数据 |
| `get_all_data` | 读 | 获取整个 Table 内容 |
| `create` | 写 | 向 Table 添加新条目 |
| `update` | 写 | 修改现有条目 |
| `delete` | 写 | 从 Table 删除条目 |

## 权限模型

权限按 Table 内的**路径**设置：

```
/                    → 可查询根级别
/products            → 可查询 products 数组
/products/*/price    → 可读取单个产品价格
```

## 示例配置

在 PuppyOne 控制台，你可以配置：

1. **只读访问** — Agent 只能查询，不能修改
2. **完全访问** — Agent 可以读写
3. **路径限定访问** — Agent 只能访问特定路径

## 最佳实践

1. **从只读开始** — 只在需要时添加写权限
2. **限定特定路径** — 不要给 Agent 访问整个 Table 的权限
3. **使用独立 Table** — 将公开/私有数据分到不同 Table
