# 本地文件夹同步

将本地文件夹与 PuppyOne 云端 Context File System 同步，使用 MUT 协议进行版本控制。

---

## 什么是本地文件夹同步

本地文件夹同步基于 MUT 协议（类似 Git 的版本控制协议），在你的本地目录和 PuppyOne 云端之间建立版本化的同步通道。

- 本地编辑文件后，`mut commit` + `mut push` 推送到云端
- 云端有变更时，`mut pull` 拉取到本地
- 每次推送都生成版本记录，可回溯、可回滚

```
┌──────────────┐                            ┌──────────────────┐
│              │  ── mut push ────────────→  │                  │
│  本地文件夹   │                             │  PuppyOne Cloud  │
│  ~/my-docs   │  ←── mut pull ───────────   │  Context FS      │
│              │                             │                  │
└──────────────┘      版本控制 (MUT)          └──────────────────┘
```

同步完成后，Agent、MCP 客户端（Cursor / Claude Desktop）和 REST API 都可以访问这些内容。

## 快速开始

### 前置条件

```bash
npm install -g puppyone    # PuppyOne CLI
pip install mutai          # MUT CLI
```

### 第一步：创建同步连接

```bash
puppyone auth login
puppyone project use "My Project"
puppyone access add filesystem docs
```

命令会输出 access endpoint URL 和 Access Key，并提示两种接入方式。

### 第二步：选一种接入方式

接下来有两条路：**Clone 全新文件夹** 还是 **接入已有文件夹**。挑你的实际场景。

#### A. Clone 全新文件夹（本地还没有文件）

```bash
mut clone <endpoint-url> --credential <access-key>
cd docs
```

会创建一个新的 `./docs/` 文件夹，把云端 scope 的内容拉下来。

#### B. 接入已有文件夹（本地已经有内容）

```bash
cd /path/to/your/existing/folder
mut connect <endpoint-url> --credential <access-key>
```

`mut connect` 会拉云端状态、和本地文件做三路合并，然后把结果推回去。**只在硬盘上的文件被上传；只在云端的文件被下载。不覆盖、不丢数据。**

> 想让 PuppyOne CLI 一步到位？用 `--link`：
>
> ```bash
> puppyone access add filesystem docs --link /path/to/your/folder
> ```
>
> 等于 `puppyone access add filesystem docs && cd <link> && mut connect ...` 一气呵成。

### 第三步：编辑、提交、推送

```bash
echo "# Welcome" > readme.md
mut commit -m "init docs"
mut push
```

## Scope（路径范围）

每个同步连接绑定一个 scope——云端项目树中的路径。不同的本地文件夹可以同步到不同的 scope：

```bash
puppyone access add filesystem docs      # scope = /docs
puppyone access add filesystem code      # scope = /code
puppyone access add filesystem config    # scope = /config
```

在云端形成统一的项目树：

```
Root/
├── docs/       ← 来自 ~/my-docs
├── code/       ← 来自 ~/my-code
└── config/     ← 来自 ~/my-config
```

这允许不同的团队成员或 Agent 各自管理自己负责的部分，最终汇合到同一个项目中。

## 同步行为

### MUT vs 老版 "Real-time" 同步

| 特性 | MUT（当前） | Real-time（已废弃） |
|------|:---:|:---:|
| 同步方式 | 手动 commit + push | 后台守护进程自动同步 |
| 版本历史 | 每次 push 产生版本 | 无版本概念 |
| 冲突处理 | 三路合并 | last-write-wins |
| 离线支持 | 支持（本地 commit） | 不支持 |
| 适用场景 | 所有场景 | 仅实时编辑 |

### 文件类型支持

| 文件类型 | Push（本地→云端） | Pull（云端→本地） |
|----------|:-:|:-:|
| 文本文件（.json / .md / .txt 等） | ✓ | ✓ |
| 二进制文件（.png / .pdf 等） | ✓ | ✓ |

## 版本管理

MUT 协议为每次 push 自动创建版本记录。

### 查看提交历史

```bash
cd ~/my-docs
mut log
```

### 回滚

在 PuppyOne Dashboard 中查看版本历史并一键回滚，或通过 API 操作。

## 配合 Claude Code 使用

本地文件夹同步是将 PuppyOne 数据交给 AI 编程工具处理的最佳方式：

1. **创建连接并接入本地文件夹**，按场景挑一种：

```bash
# A) 本地文件夹已经存在 → 一站式
puppyone access add filesystem workspace --link ~/ai-workspace

# B) 本地还没有这个目录 → 先创建 AP，再 clone 一份
puppyone access add filesystem workspace
mut clone <url> --credential <key> --dir ~/ai-workspace

# C) 本地有目录但当时忘了带 --link
cd ~/ai-workspace
mut connect <url> --credential <key>
```

2. **用 Claude Code 编辑** `~/ai-workspace` 中的文件

3. **提交推送**：

```bash
cd ~/ai-workspace
mut commit -m "AI updates"
mut push
```

所有变更都有版本历史保障，可以对比 AI 的修改、按需回滚。

## 管理连接

```bash
# 列出所有连接
puppyone access ls

# 查看 Access Key（随时可查）
puppyone access key <id>

# 重新生成 Access Key
puppyone access key <id> --regenerate

# 删除连接
puppyone access rm <id>
```

## 常见问题

### 如何同步多个文件夹？

每个文件夹创建独立的 scope 连接：

```bash
puppyone access add filesystem project-a --link ~/project-a
puppyone access add filesystem project-b --link ~/project-b
```

或者分两步：

```bash
puppyone access add filesystem project-a
mut clone   <url-a> --credential <key-a>           # 全新拉一份
# 或
cd ~/existing-project-b
mut connect <url-b> --credential <key-b>           # 接入已有文件夹
```

### Clone 还是 Connect？

| 场景 | 用什么 |
|------|--------|
| 本地还没有这个文件夹 | `mut clone` — 创建新目录，拉下云端内容 |
| 本地已经有内容，想接入云端 | `mut connect` — 三路合并，不覆盖 |
| 已有云端内容 + 本地内容，想合并 | `mut connect` — 同上，自动 merge |
| 想清空云端、推本地 | 先 `puppyone access rm <id>`，重建后用 `mut connect` |

### 推送时提示冲突怎么办？

先拉取最新版本，解决冲突后重新推送：

```bash
mut pull
# 解决冲突...
mut commit -m "resolve conflicts"
mut push
```

### Access Key 丢失了？

```bash
puppyone access key <connection-id>
```

Key 存储在数据库中，不是一次性的，随时可以再查。
