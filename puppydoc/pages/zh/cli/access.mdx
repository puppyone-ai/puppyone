# 本地文件夹同步

`puppyone access add filesystem` 创建文件夹同步连接，然后使用 `mut` CLI 进行版本控制式的同步——clone、commit、push、pull，类似 Git 的工作流。

---

## 前置条件

```bash
# 安装 PuppyOne CLI
npm install -g puppyone

# 安装 mut CLI（Python）
pip install mutai
```

---

## 完整工作流

### 1. 登录并选择项目

```bash
puppyone auth login
puppyone org use "My Organization"
puppyone project use "My Project"
```

### 2. 创建文件夹同步连接

```bash
puppyone access add filesystem docs
```

`docs` 是云端的 scope 路径——本地文件夹的内容会同步到项目树的 `/docs` 下。

命令成功后会同时输出两条接入路径：

```
  Creating filesystem access point for scope "docs"... done

  ─ Pick one: clone fresh or connect existing

    A. New folder (no local files yet):
       mut clone http://api.puppyone.ai/api/v1/mut/ap/cli_xxx --credential cli_xxx

    B. Existing folder (use the files you already have):
       cd /path/to/your/folder
       mut connect http://api.puppyone.ai/api/v1/mut/ap/cli_xxx --credential cli_xxx

  ─ Or have us run option B for you next time:
    puppyone access add filesystem docs --link /path/to/your/folder

  ─ Then commit and push:
    mut commit -m "message" && mut push

  ─ Retrieve key later:
    puppyone access key <id>
```

### 3. 选一种接入方式

接下来有两条路：**Clone 全新文件夹** 还是 **接入已有文件夹**。挑你的实际场景。

**A. Clone 全新文件夹**（本地还没有文件）：

```bash
mut clone http://api.puppyone.ai/api/v1/mut/ap/cli_xxx --credential cli_xxx
```

会在当前目录创建 `./docs/`，并拉取云端已有内容。

**B. 接入已有文件夹**（本地已经有内容）：

```bash
cd /path/to/your/existing/folder
mut connect http://api.puppyone.ai/api/v1/mut/ap/cli_xxx --credential cli_xxx
```

`mut connect` 拉云端状态、和本地文件做三路合并，然后把结果推回去。**不覆盖、不丢数据。**

> 一步到位的快捷方式：跳过第 2、3 步，让 PuppyOne CLI 帮你跑 `mut connect`。
>
> ```bash
> puppyone access add filesystem docs --link /path/to/your/folder
> ```

### 4. 本地编辑，然后提交推送

```bash
cd docs

# 正常编辑文件
echo "# API Reference" > api.md
echo '{"version": "1.0"}' > config.json

# 提交并推送到云端
mut commit -m "add api docs"
mut push
```

### 5. 拉取云端变更

当其他人（或 Agent）在云端修改了内容：

```bash
cd docs
mut pull
```

## 多文件夹同步到同一项目

你可以将多个本地文件夹同步到同一项目的不同路径下：

```bash
# 一步到位 —— 创建 AP 并把已有本地文件夹一起接入
puppyone access add filesystem docs --link ~/my-docs
puppyone access add filesystem code --link ~/my-code

# 或者拆两步：先建 AP，再 clone（全新）/ connect（已有）
puppyone access add filesystem docs
mut clone   <docs-url> --credential <docs-key>           # 全新拉一份
cd ~/existing-code && mut connect <code-url> --credential <code-key>
```

在云端，项目树结构为：

```
Root/
├── docs/          ← ./docs/ 的内容
│   ├── api.md
│   └── guide.md
└── code/          ← ./code/ 的内容
    ├── main.py
    └── utils.py
```

## 管理连接

### 列出所有连接

```bash
puppyone access ls
```

### 查看 Access Key

创建时会显示一次，之后随时可以再查：

```bash
puppyone access key <connection-id>
```

### 重新生成 Access Key

```bash
puppyone access key <connection-id> --regenerate
```

### 删除连接

```bash
puppyone access rm <connection-id>
```

## MUT 命令参考

| 命令 | 说明 |
|------|------|
| `mut clone <url> --credential <key>` | 克隆 access point 到一个**新建**文件夹 |
| `mut connect <url> --credential <key>` | 把**已有**文件夹接入 access point；三路合并、不覆盖 |
| `mut commit -m "message"` | 提交本地变更 |
| `mut push` | 推送到云端 |
| `mut pull` | 拉取云端变更 |
| `mut status` | 查看本地变更状态 |
| `mut log` | 查看提交历史 |

## 与 Claude Code / Cursor 配合

本地文件夹同步是将 PuppyOne 数据交给 AI 编程工具处理的最佳方式。三种场景，按你的情况挑：

```bash
# A) 本地已经有 workspace 文件夹 → 一站式
puppyone access add filesystem workspace --link ~/ai-workspace

# B) 本地还没有这个目录 → 创建 AP，再 clone 一份
puppyone access add filesystem workspace
mut clone <url> --credential <key>
cd workspace

# C) 已经创建了 AP 但忘了带 --link，本地又有现成目录 → 事后接入
puppyone access add filesystem workspace
cd ~/ai-workspace
mut connect <url> --credential <key>

# 用 Claude Code 或 Cursor 打开目录编辑
# AI 修改完成后，提交推送：
mut commit -m "AI updates"
mut push
```

所有变更都有版本历史，可以随时对比和回滚。

## 故障排查

### mut 命令未找到

```bash
pip install mutai
```

确保 `mut` 在 PATH 中。

### Access Key 无效

```bash
# 重新生成
puppyone access key <id> --regenerate
```

### 推送冲突

如果云端有更新的版本，先拉取再推送：

```bash
mut pull
mut commit -m "merge"
mut push
```
