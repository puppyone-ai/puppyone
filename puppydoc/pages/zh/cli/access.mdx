# 本地文件夹同步

`puppyone access add filesystem` 创建文件夹同步连接，然后使用 `mut` CLI 进行版本控制式的同步——clone、commit、push、pull，类似 Git 的工作流。

---

## 前置条件

```bash
# 安装 PuppyOne CLI
npm install -g puppyone

# 安装 mut CLI（Python）
pip install mut
```

---

## 完整工作流

### 1. 登录并选择项目

```bash
puppyone auth login
puppyone org use "My Organization"
puppyone project use "My Project"
```

### 2. 创建文件夹同步连接

```bash
puppyone access add filesystem docs
```

`docs` 是云端的 scope 路径——本地文件夹的内容会同步到项目树的 `/docs` 下。

命令成功后会输出：

```
  Creating filesystem access point for scope "docs"... done

  ─ Sync with MUT:
    mut clone http://api.puppyone.ai/api/v1/mut/ap/cli_xxx --credential cli_xxx

  ─ Then commit and push:
    mut commit -m "message" && mut push

  ─ Retrieve key later:
    puppyone access key <id>
```

### 3. Clone 到本地

复制输出中的 `mut clone` 命令：

```bash
mut clone http://api.puppyone.ai/api/v1/mut/ap/cli_xxx --credential cli_xxx
```

这会在当前目录下自动创建以 scope 路径命名的文件夹（如 `./docs/`），并拉取云端已有的内容。

### 4. 本地编辑，然后提交推送

```bash
cd docs

# 正常编辑文件
echo "# API Reference" > api.md
echo '{"version": "1.0"}' > config.json

# 提交并推送到云端
mut commit -m "add api docs"
mut push
```

### 5. 拉取云端变更

当其他人（或 Agent）在云端修改了内容：

```bash
cd docs
mut pull
```

---

## 多文件夹同步到同一项目

你可以将多个本地文件夹同步到同一项目的不同路径下：

```bash
# 创建两个 scope
puppyone access add filesystem docs
puppyone access add filesystem code

# 分别 clone
mut clone <docs-url> --credential <docs-key>
mut clone <code-url> --credential <code-key>
```

在云端，项目树结构为：

```
Root/
├── docs/          ← ./docs/ 的内容
│   ├── api.md
│   └── guide.md
└── code/          ← ./code/ 的内容
    ├── main.py
    └── utils.py
```

---

## 管理连接

### 列出所有连接

```bash
puppyone access ls
```

### 查看 Access Key

创建时会显示一次，之后随时可以再查：

```bash
puppyone access key <connection-id>
```

### 重新生成 Access Key

```bash
puppyone access key <connection-id> --regenerate
```

### 删除连接

```bash
puppyone access rm <connection-id>
```

---

## MUT 命令参考

| 命令 | 说明 |
|------|------|
| `mut clone <url> --credential <key>` | 克隆远程仓库到本地 |
| `mut commit -m "message"` | 提交本地变更 |
| `mut push` | 推送到云端 |
| `mut pull` | 拉取云端变更 |
| `mut status` | 查看本地变更状态 |
| `mut log` | 查看提交历史 |

## 与 Claude Code / Cursor 配合

本地文件夹同步是将 PuppyOne 数据交给 AI 编程工具处理的最佳方式：

```bash
# 创建连接并 clone
puppyone access add filesystem workspace
mut clone <url> --credential <key>

# 用 Claude Code 或 Cursor 打开 ./workspace/ 目录进行编辑
# AI 修改完成后，提交推送
cd workspace
mut commit -m "AI updates"
mut push
```

所有变更都有版本历史，可以随时对比和回滚。

## 故障排查

### mut 命令未找到

```bash
pip install mut
```

确保 `mut` 在 PATH 中。

### Access Key 无效

```bash
# 重新生成
puppyone access key <id> --regenerate
```

### 推送冲突

如果云端有更新的版本，先拉取再推送：

```bash
mut pull
mut commit -m "merge"
mut push
```
