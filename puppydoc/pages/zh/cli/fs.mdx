# 内容操作 (REST API)

PuppyOne 的所有内容操作通过 REST API 完成，地址为 `/api/v1/content/{project_id}/...`。文件和文件夹通过**路径**（如 `docs/readme.md`）寻址，而非 UUID。

---

## Base URL 与认证

所有请求使用以下模式：

```bash
BASE_URL="https://api.puppyone.ai/api/v1/content/{project_id}"
TOKEN="YOUR_ACCESS_TOKEN"
```

每个请求需携带认证头：

```
Authorization: Bearer $TOKEN
```

---

## 接口总览

| 接口 | 方法 | 说明 |
|------|------|------|
| `/{project_id}/ls?path=` | GET | 列出目录 |
| `/{project_id}/cat?path=` | GET | 读取文件内容 |
| `/{project_id}/stat?path=` | GET | 获取文件/目录元信息 |
| `/{project_id}/tree?path=&max_depth=` | GET | 完整目录树 |
| `/{project_id}/trash` | GET | 列出回收站 |
| `/{project_id}/write` | POST | 写入文件 |
| `/{project_id}/mkdir` | POST | 创建目录 |
| `/{project_id}/mv` | POST | 移动或重命名 |
| `/{project_id}/rm` | POST | 删除 / 移入回收站 |
| `/{project_id}/restore` | POST | 从回收站恢复 |
| `/{project_id}/bulk-write` | POST | 批量写入文件 |
| `/{project_id}/versions?path=&limit=` | GET | 版本历史 |
| `/{project_id}/version-content?path=&version=` | GET | 指定版本的内容 |
| `/{project_id}/diff?v1=&v2=` | GET | 对比两个版本 |
| `/{project_id}/rollback` | POST | 回滚到指定版本 |

---

## 浏览与读取

### 列出目录

```bash
# 列出根目录
curl "$BASE_URL/ls" \
  -H "Authorization: Bearer $TOKEN"

# 列出指定路径
curl "$BASE_URL/ls?path=docs" \
  -H "Authorization: Bearer $TOKEN"
```

### 读取文件内容

```bash
curl "$BASE_URL/cat?path=docs/readme.md" \
  -H "Authorization: Bearer $TOKEN"
```

### 获取文件或目录元信息

```bash
curl "$BASE_URL/stat?path=docs/readme.md" \
  -H "Authorization: Bearer $TOKEN"
```

### 树状浏览

```bash
# 从根目录开始的完整树
curl "$BASE_URL/tree" \
  -H "Authorization: Bearer $TOKEN"

# 指定子目录和深度限制
curl "$BASE_URL/tree?path=docs&max_depth=2" \
  -H "Authorization: Bearer $TOKEN"
```

## 创建与写入

### 创建目录

```bash
curl -X POST "$BASE_URL/mkdir" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"path": "docs/guides"}'
```

### 写入文件

写入时，如果文件不存在则创建，已存在则更新。

```bash
# 写入 JSON 文件
curl -X POST "$BASE_URL/write" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "path": "config/settings.json",
    "content": {"theme": "dark", "lang": "zh"},
    "node_type": "json",
    "message": "初始设置"
  }'

# 写入 Markdown 文件
curl -X POST "$BASE_URL/write" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "path": "docs/readme.md",
    "content": "# Hello World\n\n欢迎使用 PuppyOne。",
    "node_type": "markdown",
    "message": "添加 readme"
  }'
```

### 批量写入

一次请求写入多个文件：

```bash
curl -X POST "$BASE_URL/bulk-write" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "files": [
      {"path": "data/products.json", "content": [{"name": "Widget"}], "node_type": "json"},
      {"path": "docs/changelog.md", "content": "# Changelog", "node_type": "markdown"}
    ],
    "message": "初始化数据"
  }'
```

## 文件管理

### 移动或重命名

```bash
curl -X POST "$BASE_URL/mv" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"old_path": "docs/draft.md", "new_path": "published/article.md"}'
```

### 删除

```bash
# 软删除（移入回收站）
curl -X POST "$BASE_URL/rm" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"path": "docs/outdated.md", "permanent": false}'
```

### 查看回收站

```bash
curl "$BASE_URL/trash" \
  -H "Authorization: Bearer $TOKEN"
```

### 从回收站恢复

```bash
curl -X POST "$BASE_URL/restore" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"path": "docs/outdated.md"}'
```

## 版本管理

每次写入操作都会自动创建一个新版本（Mut commit），你可以随时查看历史、对比差异、回滚到任意版本。

### 查看版本历史

```bash
curl "$BASE_URL/versions?path=docs/readme.md&limit=10" \
  -H "Authorization: Bearer $TOKEN"
```

### 读取特定版本的内容

```bash
curl "$BASE_URL/version-content?path=docs/readme.md&version=3" \
  -H "Authorization: Bearer $TOKEN"
```

### 对比两个版本

```bash
curl "$BASE_URL/diff?v1=1&v2=3" \
  -H "Authorization: Bearer $TOKEN"
```

### 回滚到指定版本

```bash
curl -X POST "$BASE_URL/rollback" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"target_version": 2}'
```

回滚会创建一个新的 commit，内容与目标版本相同，不会丢失中间的版本记录。

## 实用工作流

### 快速搭建项目结构

```bash
# 创建目录
curl -X POST "$BASE_URL/mkdir" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"path": "docs"}'

curl -X POST "$BASE_URL/mkdir" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"path": "config"}'

# 写入初始内容
curl -X POST "$BASE_URL/write" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "path": "config/settings.json",
    "content": {"version": "1.0", "env": "production"},
    "node_type": "json",
    "message": "初始配置"
  }'
```

### 从脚本导入数据

```bash
# 将 API 返回的数据写入 PuppyOne
DATA=$(curl -s https://api.example.com/data)
curl -X POST "$BASE_URL/write" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"path\": \"imports/api-data.json\", \"content\": $DATA, \"node_type\": \"json\"}"
```

### 版本对比与回滚

```bash
# 查看版本历史
curl "$BASE_URL/versions?path=config/settings.json" \
  -H "Authorization: Bearer $TOKEN"

# 对比版本
curl "$BASE_URL/diff?v1=3&v2=5" \
  -H "Authorization: Bearer $TOKEN"

# 回滚
curl -X POST "$BASE_URL/rollback" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"target_version": 3}'
```
