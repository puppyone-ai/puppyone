# 统一连接管理

`puppyone access` 是 PuppyOne 的统一连接管理命令。所有外部集成——数据源同步、Agent、MCP 端点、沙盒、本地文件夹——都通过同一套命令管理。

---

## 命令总览

| 命令 | 说明 |
|------|------|
| `access add <type> [source]` | 创建连接 |
| `access ls` | 列出所有连接 |
| `access info <id>` | 查看连接详情 |
| `access pause <id>` | 暂停连接 |
| `access resume <id>` | 恢复连接 |
| `access rm <id>` | 删除连接 |
| `access key <id>` | 查看 Access Key |
| `access refresh <id>` | 手动触发同步 |
| `access trigger <id> <mode>` | 设置触发模式 |
| `access update <id>` | 更新配置 |
| `access logs <id>` | 查看执行记录 |
| `access run <id>` | 触发执行 |
| `access schema <provider>` | 查看配置字段 |

---

## 创建连接

`access add` 是创建连接的入口，第一个参数指定连接类型，后续参数和选项因类型而异。

### 连接类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `notion` | Notion 数据源 | `access add notion <url>` |
| `github` | GitHub 仓库 | `access add github <url>` |
| `gmail` | Gmail 邮箱 | `access add gmail` |
| `url` | 网页抓取 | `access add url <url>` |
| `agent` | AI Agent | `access add agent "name"` |
| `mcp` | MCP 端点 | `access add mcp "name"` |
| `sandbox` | 代码沙盒 | `access add sandbox "name"` |
| `filesystem` | 本地文件夹同步 | `access add filesystem <scope>` |

### 通用选项

| 选项 | 说明 |
|------|------|
| `--name <name>` | 连接名称 |
| `--folder <path>` | 数据存放的云端目录 |
| `--mode <mode>` | 同步模式 |
| `--model <model>` | Agent 使用的模型 |
| `--system-prompt <text>` | Agent 的系统提示词 |
| `--type <type>` | 沙盒类型（`e2b` / `docker`） |
| `--api-key <key>` | API Key |
| `--config <json>` | 高级配置（JSON 格式） |

### 数据源连接

从外部平台拉取数据到你的 Content Node 树：

```bash
# Notion — 同步一个页面
puppyone access add notion https://notion.so/my-page-id --folder /docs

# GitHub — 同步一个仓库
puppyone access add github https://github.com/org/repo --folder /code

# Gmail — 同步邮件
puppyone access add gmail

# 网页抓取
puppyone access add url https://example.com/article --folder /imports
```

### Agent 连接

创建一个 AI Agent，绑定模型和工具：

```bash
# 创建一个 GPT-4o Agent
puppyone access add agent "My Bot" --model gpt-4o

# 创建带系统提示词的 Agent
puppyone access add agent "Analyst" --model gpt-4o --system-prompt "你是一个数据分析师"
```

### MCP 端点

创建 MCP 端点，让 Cursor、Claude Desktop 等客户端直接访问你的数据：

```bash
puppyone access add mcp "My Endpoint"
```

创建后使用 `access key` 获取 Access Key，配置到客户端中。

### 沙盒

创建隔离的代码执行环境：

```bash
# 默认沙盒
puppyone access add sandbox "Runner"

# 指定 E2B 类型
puppyone access add sandbox "Runner" --type e2b
```

### 本地文件夹同步

将本地目录通过 MUT 协议同步到云端，两种典型场景：

```bash
# A) 一站式 —— 已有本地文件夹，直接接入
puppyone access add filesystem docs --link ~/my-docs

# B) 仅创建云端 access point，事后再接入
puppyone access add filesystem docs

# 然后从 access add 的输出里挑一条命令跑：
#   云端有内容、本地还没目录 → 新建文件夹拉一份
mut clone <url> --credential <key>

#   本地已经有内容 → 接入，三路合并不覆盖
cd ~/my-docs
mut connect <url> --credential <key>
```

详见 [本地文件夹同步](/zh/cli/access) 文档。

## 查看与管理

### 列出所有连接

```bash
# 列出所有连接
puppyone access ls

# 按类型筛选
puppyone access ls --provider notion

# 按状态筛选
puppyone access ls --status active

# JSON 输出
puppyone access ls --json
```

### 查看连接详情

```bash
puppyone access info conn_abc123
```

### 暂停与恢复

```bash
# 暂停一个数据源同步
puppyone access pause conn_abc123

# 恢复
puppyone access resume conn_abc123
```

### 删除连接

```bash
puppyone access rm conn_abc123
```

---

## Access Key 管理

Agent、MCP 端点和沙盒都使用 Access Key 进行身份验证。

```bash
# 查看 Access Key
puppyone access key conn_abc123

# 重新生成 Access Key
puppyone access key conn_abc123 --regenerate
```

---

## 同步与执行

### 手动触发同步

对数据源连接，手动执行一次同步：

```bash
puppyone access refresh conn_abc123
```

### 设置触发模式

配置连接的同步触发方式：

```bash
puppyone access trigger conn_abc123 manual
puppyone access trigger conn_abc123 scheduled
```

### 触发执行

直接运行一次连接的执行逻辑：

```bash
puppyone access run conn_abc123
```

### 查看执行记录

```bash
puppyone access logs conn_abc123
```

---

## 更新配置

修改已有连接的配置：

```bash
# 更新单个配置项
puppyone access update conn_abc123 --set name="New Name"

# 通过 JSON 更新多项配置
puppyone access update conn_abc123 --config '{"folder": "/new-path", "mode": "incremental"}'
```

### 查看配置字段

查看某种连接类型支持哪些配置字段：

```bash
puppyone access schema notion
puppyone access schema agent
```

## 实用工作流

### 为项目搭建多数据源

```bash
# 连接 Notion 文档
puppyone access add notion https://notion.so/wiki-page --folder /wiki

# 连接 GitHub 仓库
puppyone access add github https://github.com/org/docs --folder /github

# 连接 Gmail
puppyone access add gmail

# 查看所有连接状态
puppyone access ls
```

### 创建并使用 MCP 端点

```bash
# 创建端点
puppyone access add mcp "Cursor Endpoint"

# 获取 Access Key
puppyone access key <id>

# 在 Cursor 的 MCP 配置中使用该 Key 连接
```

### 创建并管理沙盒

```bash
# 创建沙盒
puppyone access add sandbox "Code Runner" --type e2b

# 查看详情
puppyone access info <id>

# 获取 Access Key
puppyone access key <id>

# 触发一次执行
puppyone access run <id>

# 查看执行日志
puppyone access logs <id>
```

### 批量管理连接

```bash
# 列出所有暂停的连接
puppyone access ls --status paused --json

# 通过脚本批量恢复
for id in $(puppyone access ls --status paused --json | jq -r '.[].id'); do
  puppyone access resume "$id"
done
```
