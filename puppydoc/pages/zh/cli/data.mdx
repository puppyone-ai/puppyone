# 远程文件系统操作

`puppyone data` 让你从终端直接操作 PuppyOne 云端 workspace 中的文件和文件夹——浏览、读取、写入、移动、删除，无需打开浏览器。

这是 Web UI 中看到的同一棵内容树，现在可以从任何终端、脚本或 AI Agent 直接访问。

---

## 命令总览

| 命令 | 说明 |
|------|------|
| `data ls [path]` | 列出目录内容 |
| `data cat <path>` | 读取文件内容 |
| `data tree [path]` | 显示目录树 |
| `data stat <path>` | 查看文件/目录元信息 |
| `data write <path>` | 写入文件 |
| `data touch <path>` | 创建空文件 |
| `data mkdir <path>` | 创建目录 |
| `data cp <src> <dst>` | 复制文件 |
| `data mv <src> <dst>` | 移动或重命名 |
| `data rm <path>` | 删除（默认移入回收站） |
| `data trash` | 查看回收站 |
| `data restore <path>` | 从回收站恢复 |

所有命令都支持 `--json` 输出。

---

## 浏览与读取

### 列出目录

```bash
# 列出根目录
puppyone data ls

# 列出指定目录
puppyone data ls /docs

# 详细模式（显示类型、大小、hash）
puppyone data ls /docs -l
```

### 读取文件

```bash
puppyone data cat /docs/readme.md

# JSON 文件会自动格式化输出
puppyone data cat /config/settings.json
```

### 目录树

```bash
# 完整目录树
puppyone data tree

# 指定起点和深度
puppyone data tree /docs --depth 2
```

### 文件信息

```bash
puppyone data stat /docs/readme.md
```

输出包括路径、类型、大小、MIME 类型、内容 hash 等。

## 创建与写入

### 写入文件

`write` 命令支持三种内容来源：内联文本、本地文件、stdin 管道。

```bash
# 内联写入 JSON
puppyone data write /config.json --content '{"theme": "dark"}'

# 从本地文件读取内容写入云端
puppyone data write /docs/guide.md --file ~/local-guide.md

# 从管道写入
echo "Hello World" | puppyone data write /docs/hello.md

# 指定 commit message
puppyone data write /config.json --content '{}' -m "reset config"
```

文件类型根据扩展名自动检测（`.json` → json，`.md` → markdown），也可以用 `--type` 手动指定。

### 创建空文件

```bash
puppyone data touch /docs/draft.md
puppyone data touch /data/empty.json
```

### 创建目录

```bash
puppyone data mkdir /docs/guides
```

---

## 文件管理

### 复制

```bash
puppyone data cp /docs/readme.md /backup/readme.md
```

### 移动或重命名

```bash
puppyone data mv /docs/old-name.md /docs/new-name.md
```

### 删除

```bash
# 默认移入回收站（可恢复）
puppyone data rm /docs/outdated.md

# 永久删除
puppyone data rm /docs/outdated.md --force
```

### 回收站

```bash
# 查看回收站内容
puppyone data trash

# 恢复文件
puppyone data restore .trash/docs/outdated.md
```

## 与脚本和 AI Agent 集成

所有命令支持 `--json` 输出，方便自动化：

```bash
# 列出所有文件名
puppyone data ls /docs --json | jq '.entries[].name'

# 读取 JSON 文件并提取字段
puppyone data cat /config.json --json | jq '.content.theme'

# 将外部 API 数据写入 PuppyOne
curl -s https://api.example.com/data | puppyone data write /imports/api-data.json
```

### 在 CI/CD 中使用

```bash
# 使用 API Key 认证（无需交互式登录）
puppyone data ls / --api-key $PUPPYONE_KEY --project $PROJECT_ID --json
```

## 与 REST API 的关系

`puppyone data` 命令是 PuppyOne [内容操作 REST API](/zh/cli/fs) 的 CLI 封装。两者操作同一棵内容树，选择哪种取决于使用场景：

- **CLI**：终端操作、脚本、AI 编程工具集成
- **REST API**：应用代码、自定义集成、批量操作（如 `bulk-write`）、版本管理（`versions`、`diff`、`rollback`）

版本管理相关操作（查看历史、对比版本、回滚）目前仅通过 REST API 提供，详见 [文件系统命令](/zh/cli/fs) 文档。
