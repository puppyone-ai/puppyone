# CLI 安装与配置

PuppyOne CLI（`puppyone`）是一个命令行工具，让你从终端直接操作 PuppyOne 平台——项目管理、Access Point 配置、Agent 对话等。

---

## 安装

通过 npm 全局安装：

```bash
npm install -g puppyone
```

安装完成后，验证版本：

```bash
puppyone -V
```

---

## 首次配置

安装后需要完成三步初始化：登录、选择组织、选择项目。

```bash
# 1. 登录
puppyone auth login -e user@example.com -p password

# 2. 选择组织
puppyone org use "My Organization"

# 3. 选择项目
puppyone project use "My Project"
```

完成后，CLI 会将当前状态保存到配置文件中，后续命令自动使用已选择的组织和项目。

---

## 全局选项

所有命令都支持以下全局选项：

| 选项 | 说明 |
|------|------|
| `-V, --version` | 显示版本号 |
| `-u, --api-url <url>` | 指定 API 地址（覆盖配置文件） |
| `-k, --api-key <key>` | 指定 API Key / Token（覆盖配置文件） |
| `--json` | 以 JSON 格式输出（适合脚本和 AI 消费） |
| `-v, --verbose` | 显示详细输出 |
| `-p, --project <id>` | 指定项目 ID（覆盖当前活动项目） |
| `-o, --org <id>` | 指定组织 ID（覆盖当前活动组织） |

示例：临时切换到另一个项目执行命令，无需修改全局配置：

```bash
puppyone access ls -p proj_abc123
```

---

## JSON 输出

所有命令都支持 `--json` 选项，输出机器可读的 JSON 格式，方便与其他工具或 AI Agent 集成：

```bash
puppyone access ls --json
```

---

## 配置文件

CLI 的配置保存在 `~/.puppyone/config.json`，包含 API 地址、登录凭证、当前活动的组织和项目等信息。

### 配置命令

```bash
# 查看当前配置
puppyone config show

# 设置配置项
puppyone config set api_url https://api.puppyone.ai

# 查看配置文件路径
puppyone config path

# 重置配置
puppyone config reset
```

## 命令一览

PuppyOne CLI 将功能划分为以下命令组：

| 命令 | 说明 |
|------|------|
| `puppyone auth` | 登录、登出、查看当前用户 |
| `puppyone org` | 组织管理 |
| `puppyone project` | 项目管理 |
| `puppyone access` | 统一 Access Point 管理（数据源、Agent、MCP、沙盒、文件夹同步） |
| `puppyone chat` | Agent 对话 |
| `puppyone config` | CLI 配置管理 |
| `puppyone status` | 项目仪表盘 |
| `puppyone ps` | 查看正在运行的守护进程 |

内容操作（浏览、读写、版本管理）通过 [REST API](/zh/cli/fs) 完成，而非 CLI 命令。

## 常用工作流

### 添加数据源并同步

```bash
puppyone access add notion https://notion.so/page-id --folder /notion-data
puppyone access ls
```

### 创建 Agent 并对话

```bash
puppyone access add agent "My Bot" --model gpt-4o
puppyone chat
```

### 创建 MCP 端点

```bash
puppyone access add mcp "My Data"
```

### 挂载本地文件夹

```bash
puppyone access add filesystem ~/workspace
```

### 查看项目状态

```bash
puppyone status
puppyone access ls
```
