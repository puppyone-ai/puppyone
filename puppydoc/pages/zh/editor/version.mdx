# 版本历史

PuppyOne 自动追踪每次数据变更，支持查看历史和回滚。

---

## 自动版本控制

每次修改 Table 数据时，系统会自动：

1. 保存当前版本快照
2. 记录修改时间和操作者
3. 保留历史版本供回滚

---

## 查看历史

### 打开版本面板

1. 在 Table 编辑器中
2. 点击工具栏的 **History** 按钮
3. 右侧展开版本历史面板

### 版本列表

```
📋 版本历史
├── v12 - 2024-01-20 15:30 (当前)
│   └── "更新了产品价格"
├── v11 - 2024-01-20 14:00
│   └── "添加了新产品"
├── v10 - 2024-01-19 10:00
│   └── "从 Notion 同步"
└── ...
```

---

## 对比版本

1. 选择要对比的两个版本
2. 点击 **Compare**
3. 查看差异：
   - 🟢 绿色：新增
   - 🔴 红色：删除
   - 🟡 黄色：修改

```diff
{
  "products": [
    {
      "name": "Widget Pro",
-     "price": 99.99,
+     "price": 89.99,
    }
  ]
}
```

---

## 回滚版本

### 完整回滚

1. 选择要恢复的版本
2. 点击 **Restore**
3. 确认回滚

> ⚠️ 回滚会覆盖当前数据，但当前版本仍会保留在历史中。

### 部分恢复

1. 查看历史版本
2. 复制需要的内容
3. 手动粘贴到当前版本

---

## 版本保留策略

| 计划 | 保留时间 | 版本数量 |
|------|----------|----------|
| Free | 7 天 | 最近 20 个 |
| Pro | 30 天 | 最近 100 个 |
| Team | 90 天 | 无限制 |

---

## 最佳实践

### 1. 大改动前先标记

在进行大规模修改前，添加一个描述性的版本注释：

1. 点击 **Save with Note**
2. 输入描述（如「重构产品分类结构」）

### 2. 定期检查

建议每周检查一次版本历史，清理不需要的实验性版本。

### 3. 导出备份

对于重要数据，定期导出 JSON 备份：

1. 点击 **Export** → **JSON**
2. 保存到本地
