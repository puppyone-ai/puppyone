# 管理数据

在 PuppyOne 中编辑、组织你的结构化数据。

---

## 为什么需要管理数据

导入的原始数据可能需要：

- **清理**：删除无关字段、修正错误
- **重组**：调整 JSON 结构，便于 Agent 查询
- **补充**：添加元数据、标签

---

## PuppyOne 数据模型

```
Workspace（工作区）
└── Project（项目）
    └── Table（表）
        └── JSON Data（结构化数据）
```

| 概念 | 说明 |
|------|------|
| **Project** | 顶层容器，一个业务场景对应一个 Project |
| **Table** | Project 内的数据表，存储一份结构化 JSON |
| **JSON Data** | 实际数据，可以是对象、数组、嵌套结构 |

## 数据结构建议

为了让 Agent 更高效地使用数据，建议：

### ✅ 推荐的结构

```json
{
  "products": [
    {
      "id": "SKU-001",
      "name": "Widget Pro",
      "price": 99.99,
      "category": "电子产品"
    }
  ],
  "categories": ["电子产品", "配件", "服务"]
}
```

**特点**：扁平化、字段名清晰、类型一致

### ❌ 不推荐的结构

```json
{
  "data": {
    "items": {
      "group1": {
        "subgroup": {
          "products": [...]
        }
      }
    }
  }
}
```

**问题**：嵌套过深，Agent 难以准确定位
