# OpenClaw

把 puppyone 云端的文件夹实时同步到你的本地机器，让 Claude Code 等 Agent 直接操作文件。

**接入方式：文件同步（双向，实时）**

---

## 配置步骤

### Step 1：在 Dashboard 创建 OpenClaw Agent

1. 打开你的 Project → 侧边栏点击 **新建 Agent**
2. 选择类型 **OpenClaw**
3. 在 **Workspace folder** 区域，把左侧文件树中的一个**文件夹**拖入（只支持文件夹，最多 1 个）
4. 点击 **Save** 创建

创建成功后，Agent 面板显示 **"Waiting for CLI"**，等待本地 CLI 连接。

---

### Step 2：安装 puppyone CLI

```bash
npm install -g puppyone
```

---

### Step 3：运行同步命令

在 Agent 面板点击 **Connect**，弹出配置向导，复制生成的命令：

```bash
puppyone access up \
  --key cli_xxxxxxxx \
  -u https://api.puppyone.ai \
  --path /Users/yourname/.openclaw/workspace
```

| 参数 | 说明 |
|------|------|
| `--key` | Access Key（在 Dashboard 中生成，格式 `cli_xxx`） |
| `-u` | puppyone API 地址 |
| `--path` | 本地同步文件夹路径（不存在会自动创建） |

运行后，CLI 后台守护进程启动，终端可以关闭。

---

### Step 4：确认连接状态

回到 Dashboard，Agent 面板状态变为 **"Sync active"**（绿色），说明同步已建立。

```
OpenClaw ←──────────────────────────────────► puppyone
本地文件夹                实时双向同步               云端文件夹
```

---

## 使用方式

同步建立后，直接用 Claude Code 或任意 Agent 操作本地文件夹：

```
我的 context 文件在 ~/.openclaw/workspace 目录下。
请读取里面的文档，帮我完成...
```

对本地文件的任何修改都会自动同步到 puppyone 云端；云端有新数据（外部数据源更新、其他 Agent 写入）也会自动同步到本地。

---

## 状态说明

| 状态 | 含义 |
|------|------|
| **Waiting for CLI** | CLI 尚未运行，等待连接 |
| **Sync active** | 守护进程在线，同步正常 |
| **Daemon offline** | 曾经连接过，但守护进程已停止 |

如果状态变为 Daemon offline，重新运行 `puppyone access up` 命令即可恢复。

---

## 注意事项

- 每个 OpenClaw Agent 只能绑定一个文件夹
- Access Key 与文件夹绑定，不同文件夹需要分别创建 Agent
- 支持的文件类型：所有文本文件（Markdown、JSON、代码）；二进制文件目前只支持 Pull
