# Claude Code

把 puppyone 的 Context File System 同步到本地文件夹，让 Claude Code 像操作本地项目一样使用你的数据。

**接入方式：文件同步（双向）**

---

## 工作原理

Claude Code 的核心工作方式是操作本地文件和执行 Bash 命令。puppyone 通过 OpenClaw 把云端的 Context File System 实时同步到你的本地文件夹：

```
puppyone 云端                      本地文件夹
┌──────────────────┐               ┌──────────────────────┐
│ /notion-docs/    │  ◄──────────► │ ~/project/context/   │
│ /github-issues/  │   实时双向同步  │   /notion-docs/      │
│ /linear-tasks/   │               │   /github-issues/    │
└──────────────────┘               └──────────┬───────────┘
                                              │
                                   Claude Code 在这里读写文件
```

Claude Code 修改文件 → 自动同步回云端，云端有更新 → 自动同步到本地。

---

## 配置步骤

### Step 1：安装 OpenClaw

```bash
npm install -g openclaw
```

### Step 2：获取 Access Key

1. 打开 puppyone Dashboard → 你的 Project
2. 点击 **Connect Agent** → **Claude Code**
3. 复制生成的 Access Key

### Step 3：连接 Project

```bash
openclaw connect \
  --key YOUR_ACCESS_KEY \
  --local ~/project/context
```

### Step 4：启动同步

```bash
openclaw start
```

守护进程启动后，本地文件夹和云端保持实时同步。

---

## 开始使用

同步启动后，直接在 Claude Code 中引用本地文件夹：

```
我的 context 文件在 ~/project/context 目录下。
请阅读 /notion-docs/product-spec.md，然后根据 /linear-tasks/ 里的任务列表，
帮我更新 /specs/roadmap.md
```

Claude Code 对本地文件的任何修改都会自动同步回 puppyone 云端。

---

## 为什么不用 MCP？

Claude Code 是文件系统原生的 Agent——它更擅长读写文件，而不是调用 API。文件同步比 MCP 更符合 Claude Code 的工作方式：

- 不需要每次查询都调用 API
- Claude Code 可以用 Bash 批量处理文件
- 改动有完整的本地文件历史，也有云端版本记录
