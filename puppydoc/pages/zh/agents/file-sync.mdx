# 文件同步

把 Context File System 同步到本地文件夹，让 Agent 像操作本地项目一样使用你的数据。

---

## 工作原理

```
puppyone 云端                       本地
┌─────────────────────┐             ┌─────────────────────┐
│  Context File System │             │  本地文件夹          │
│  /notion-docs/       │  ◄──────►  │  /project/context/  │
│  /github-issues/     │   双向同步   │  /notion-docs/      │
│  /specs/             │  (实时)     │  /github-issues/    │
└─────────────────────┘             └──────────┬──────────┘
                                               │
                                    Agent 在这里读写文件
                                    (Claude Code / OpenClaw)
```

同步是**双向的、实时的**：
- Agent 在本地修改文件 → 自动同步到云端
- 云端有新数据（外部数据源更新、其他 Agent 写入）→ 自动同步到本地

---

## OpenClaw 文件同步

OpenClaw 是 puppyone 官方的文件同步客户端，专为 Claude Code 等新一代 Agent 设计。

### 工作方式

1. 在本地安装 OpenClaw CLI
2. 运行 `openclaw connect`，用 Access Key 绑定你的 puppyone Project
3. OpenClaw 启动一个后台守护进程，监听文件变化
4. Claude Code（或任意 Agent）操作本地文件夹，改动实时双向同步

### 配置步骤

**Step 1：安装 OpenClaw**

```bash
npm install -g openclaw
# 或
brew install openclaw
```

**Step 2：获取 Access Key**

1. 打开 puppyone Dashboard → 你的 Project
2. 点击 **Connect Agent** → **OpenClaw**
3. 复制生成的 Access Key

**Step 3：连接 Project**

```bash
openclaw connect \
  --key YOUR_ACCESS_KEY \
  --local /path/to/your/local/folder
```

**Step 4：启动同步守护进程**

```bash
openclaw start
```

守护进程启动后，本地文件夹和云端保持实时双向同步。

### 给 Claude Code 使用

一旦同步启动，直接把本地文件夹路径告诉 Claude Code：

```
在 /path/to/your/local/folder 下有我的所有 context 文件，
帮我分析 /notion-docs 里的产品文档，并更新 /specs/roadmap.md
```

Claude Code 修改 `roadmap.md` 后，改动会自动同步回 puppyone 云端。

## 本地文件夹直接同步

除了 OpenClaw，你也可以直接把本地文件夹（不通过 OpenClaw CLI）双向挂载到 puppyone。

### 适用场景

- 你已经有一个本地项目文件夹，想让它和 puppyone 保持同步
- 多个 Agent 共享同一份本地文件

### 配置步骤

1. 打开 puppyone Dashboard → 你的 Project
2. 点击 **Add Connector** → **Local Folder**
3. 选择本地文件夹路径
4. 点击 **Start Sync**

## 常见问题

**Q：双向同步有冲突怎么处理？**

puppyone 使用基于时间戳的冲突解决策略：最后写入的版本胜出。所有版本都保留在版本历史中，可随时回滚。

**Q：本地文件夹断网了怎么办？**

OpenClaw 在本地缓存所有文件。断网期间的改动会在重新联网后自动同步。

**Q：哪些文件类型支持双向同步？**

Markdown、JSON、代码文件（.py/.ts/.js 等）、纯文本——所有文本文件都支持。二进制文件（图片、PDF）目前只支持单向同步（Pull）。
