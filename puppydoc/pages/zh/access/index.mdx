# File Level Security (FLS)

精确控制每个 Agent 能看到什么、能做什么操作。

---

## 什么是 FLS

**File Level Security（FLS）** 是 puppyone 的权限控制系统。

类比数据库的行级权限（Row Level Security），FLS 把权限精确到文件路径级别。每个 API Key 对应一组权限规则，决定：

1. **Tool 权限**：这个 Key 能调用哪些操作（查询 / 创建 / 修改 / 删除）
2. **路径权限**：每个操作能访问哪些 JSON 路径

没有被授权的路径，**对 Agent 来说物理上不存在**——它看到的文件系统就是一个干净的、只包含它能访问的内容的视图。

---

## 两层权限模型

```
┌──────────────────────────────────────┐
│           Tool 权限                   │
│  Agent 能调用哪些操作？               │
│  query_data ✅  create ✅  delete ❌  │
└──────────────────────────────────────┘
                  │
                  ▼
┌──────────────────────────────────────┐
│           路径权限                    │
│  每个操作能访问哪些路径？             │
│  /products ✅  /internal ❌          │
└──────────────────────────────────────┘
```

---

## Tool 权限

| Tool | 能力 | 风险等级 |
|------|------|----------|
| `get_data_schema` | 获取数据结构 | 🟢 低 |
| `get_all_data` | 读取全部数据 | 🟢 低 |
| `query_data` | 带条件过滤查询 | 🟢 低 |
| `preview` | 快速预览部分数据 | 🟢 低 |
| `select` | 返回指定字段 | 🟢 低 |
| `create` | 创建新条目 | 🟡 中 |
| `update` | 修改现有条目 | 🟡 中 |
| `delete` | 删除条目 | 🔴 高 |

---

## 路径权限

使用 JSON Path 语法指定允许或禁止的路径：

```
/                    → 整个数据（根路径）
/products            → products 数组
/products/*/price    → 所有产品的价格字段
/internal            → internal 路径（可设为禁止）
```

---

## 配置示例

### 客服 Agent（只读）

```yaml
Tools:
  - get_data_schema: ✅
  - get_all_data:    ✅
  - query_data:      ✅
  - preview:         ✅
  - create:          ❌
  - update:          ❌
  - delete:          ❌

Paths:
  /products: 允许
  /faq:      允许
  /internal: 禁止
```

### 研发 Agent（读写，禁止删除）

```yaml
Tools:
  - query_data: ✅
  - create:     ✅
  - update:     ✅
  - delete:     ❌

Paths:
  /specs:     允许
  /code_docs: 允许
```

### 管理员 Agent

```yaml
Tools: 全部允许
Paths: /（根路径，无限制）
```

---

## 配置方式

1. 打开 Project → **Access** 标签页
2. 选择一个 API Key（或新建一个）
3. 勾选允许的 Tool
4. 点击 **Add Path Rule** 配置路径权限
5. 保存
