# PuppyOne

> **为 AI Agent 而生的 Context Base。**

PuppyOne 是专为 AI Agent 设计的**结构化上下文平台**。  
它解决一个核心问题：**让你的 Agent 在精确数据上可靠工作，而非模糊检索后产生幻觉。**

## 为什么选择 PuppyOne？

| 传统方案 | 问题 | PuppyOne 的解法 |
|---------|------|----------------|
| 知识库 (Notion, 飞书) | 为人类设计，格式混乱 | Agent-native 结构化索引 |
| 向量数据库 | 概率性检索，SKU/价格等精确数据易幻觉 | 确定性 JSON 存取 |
| 手动维护 JSON | 无版本控制、无权限、无分发 | 完整的上下文生命周期管理 |

## 核心能力

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│    导入     │ →  │    结构化   │ →  │    索引     │ →  │    分发     │
├─────────────┤    ├─────────────┤    ├─────────────┤    ├─────────────┤
│ PDF, Notion │    │ 可视化 JSON │    │ 向量 +      │    │ MCP 协议    │
│ Airtable    │    │ 树形编辑器  │    │ 语义 +      │    │ REST API    │
│ 网页爬取    │    │             │    │ 精确路径    │    │ 公开链接    │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

1. **导入 (Import)** — 从 PDF、Notion、Airtable、网页等导入数据，自动 ETL 清洗
2. **结构化 (Structure)** — 可视化 JSON 编辑器，维护干净有序的上下文
3. **索引 (Index)** — 混合索引：向量 + 语义 + 确定性路径查询
4. **分发 (Distribute)** — 通过 MCP 协议、REST API 或公开链接分发给 Agent
