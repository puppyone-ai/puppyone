# Path Permissions

Restrict agent access to specific JSON paths within your data.

---

## JSON Path Syntax

| Pattern | Matches |
|---------|---------|
| `/` | Root — the entire dataset |
| `/products` | The `products` field |
| `/products/0` | First element of products array |
| `/products/*` | All direct children of products |
| `/products/**` | All nested descendants of products |

---

## How Rules Work

Rules are evaluated in order. The first matching rule wins.

```yaml
Rules:
  /products: allow        # Allow access to products
  /products/*/cost: deny  # But block the cost field
  /internal: deny         # Block internal data
  /: allow                # Allow everything else
```

If no rule matches a path, the default is **deny**.

---

## Common Scenarios

### Read-only product catalog (hide pricing)

```yaml
/products: allow
/products/*/price: deny
/products/*/cost: deny
```

### Engineering docs only

```yaml
/specs: allow
/code_docs: allow
/: deny
```

### Everything except internal

```yaml
/internal: deny
/: allow
```
