# Tool Permissions

Control which operations each agent API Key can invoke.

---

## Available Tools

| Tool | Default Name | Description | Risk |
|------|-------------|-------------|------|
| `get_data_schema` | `get_data_schema` | Returns the data structure definition | 🟢 Read |
| `get_all_data` | `get_all_data` | Returns the full dataset | 🟢 Read |
| `query_data` | `query_data` | Filtered query with conditions | 🟢 Read |
| `preview` | `preview_data` | Returns the first N records | 🟢 Read |
| `select` | `select_data` | Returns only specified fields | 🟢 Read |
| `create` | `create_element` | Adds a new entry | 🟡 Write |
| `update` | `update_element` | Modifies an existing entry | 🟡 Write |
| `delete` | `delete_element` | Removes an entry | 🔴 Destructive |

---

## Best Practices

**1. Start read-only.** Grant write tools only when the agent explicitly needs them.

**2. Never grant `delete` by default.** Deletes are irreversible — require intentional authorization.

**3. Use separate API Keys per agent.** A customer support agent shouldn't share a key with a dev agent that has write access.

**4. Audit regularly.** Review which keys have write or delete permissions periodically.
