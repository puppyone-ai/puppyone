# File Sync

Sync the Context File System to a local folder so agents can work with files natively — no API calls required.

---

## How It Works

```
PuppyOne Cloud                        Local Folder
┌─────────────────────┐               ┌─────────────────────┐
│  Context File System │               │  /project/context/  │
│  /notion-docs/       │  ◄──────────► │  /notion-docs/      │
│  /github-issues/     │   2-way sync  │  /github-issues/    │
│  /specs/             │   (realtime)  │  /specs/            │
└─────────────────────┘               └──────────┬──────────┘
                                                 │
                                      Agent reads & writes here
                                      (Claude Code / OpenClaw)
```

Sync is **bidirectional and real-time**:
- Agent modifies a local file → automatically synced to cloud
- Cloud data updates (new connector sync, another agent's write) → automatically synced to local

## OpenClaw File Sync

OpenClaw is PuppyOne's official file sync client, built for Claude Code and next-generation file-system-native agents.

### How It Works

1. Install the OpenClaw CLI locally
2. Run `openclaw connect` with an Access Key to link your PuppyOne Project
3. OpenClaw starts a background daemon that watches for file changes
4. Claude Code (or any agent) operates on the local folder — changes sync in real time

### Setup

**Step 1: Install OpenClaw**

```bash
npm install -g openclaw
# or
brew install openclaw
```

**Step 2: Get Your Access Key**

1. Open PuppyOne Dashboard → your Project
2. Click **Connect Agent** → **OpenClaw**
3. Copy the generated Access Key

**Step 3: Connect Your Project**

```bash
openclaw connect \
  --key YOUR_ACCESS_KEY \
  --local /path/to/your/local/folder
```

**Step 4: Start the Sync Daemon**

```bash
openclaw start
```

Once running, the local folder and cloud stay in real-time sync.

### Using with Claude Code

Point Claude Code to the local folder:

```
In /path/to/your/local/folder I have all my context files.
Analyze the product docs in /notion-docs and update /specs/roadmap.md
```

When Claude Code edits `roadmap.md`, the change syncs back to PuppyOne automatically.

## Local Folder Sync (without OpenClaw)

You can also sync a local folder directly to PuppyOne without the OpenClaw CLI — for example, if you already have an existing project folder.

### Setup

1. Open PuppyOne Dashboard → your Project
2. Click **Add Connector** → **Local Folder**
3. Select the local folder path
4. Click **Start Sync**

## FAQ

**What happens if there's a sync conflict?**

PuppyOne uses a last-write-wins strategy based on timestamps. All versions are preserved in version history and can be rolled back.

**What if the local machine goes offline?**

OpenClaw caches all files locally. Changes made while offline are synced automatically when the connection is restored.

**Which file types support bidirectional sync?**

Markdown, JSON, code files (`.py`, `.ts`, `.js`, etc.), plain text — all text files are fully supported. Binary files (images, PDFs) are currently pull-only.
