# Claude Code

Sync PuppyOne's Context File System to a local folder so Claude Code can work with your data like a local project.

**Connection method: File sync (bidirectional)**

---

## How It Works

Claude Code's native way of working is reading and writing local files, running Bash commands. PuppyOne uses OpenClaw to keep the cloud Context File System in real-time sync with a local folder:

```
PuppyOne Cloud                      Local Folder
┌──────────────────┐                ┌──────────────────────┐
│ /notion-docs/    │  ◄───────────► │ ~/project/context/   │
│ /github-issues/  │   2-way sync   │   /notion-docs/      │
│ /linear-tasks/   │   (realtime)   │   /github-issues/    │
└──────────────────┘                └──────────┬───────────┘
                                               │
                                    Claude Code reads & writes here
```

Claude Code edits a file → automatically synced to cloud. Cloud gets an update → automatically synced to local.

## Setup

### Step 1: Install OpenClaw

```bash
npm install -g openclaw
```

### Step 2: Get Your Access Key

1. Open PuppyOne Dashboard → your Project
2. Click **Connect Agent** → **Claude Code**
3. Copy the generated Access Key

### Step 3: Connect Your Project

```bash
openclaw connect \
  --key YOUR_ACCESS_KEY \
  --local ~/project/context
```

### Step 4: Start Syncing

```bash
openclaw start
```

The daemon runs in the background, keeping local and cloud in sync.

---

## Start Using

Once sync is running, point Claude Code at the local folder:

```
My context files are in ~/project/context.
Read /notion-docs/product-spec.md and the tasks in /linear-tasks/,
then update /specs/roadmap.md accordingly.
```

Any file Claude Code modifies is automatically synced back to PuppyOne.

---

## Why not MCP?

Claude Code is a file-system-native agent — it's built to read and write files, not call APIs. File sync is a better fit than MCP for Claude Code:

- No API calls needed for every read
- Claude Code can batch-process files with Bash
- Local file history plus cloud version history for all changes
