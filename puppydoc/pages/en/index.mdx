# puppyone

**The Context File System for AI Agents.**

Sync your scattered data into a structured file system that agents can read and write directly.

---

## Everything is Sync

puppyone has one core concept: **Synchronize**.

Whether it's data being pulled from Notion, an agent reading and writing via MCP, or a local folder staying in sync via OpenClaw — it's all the same sync engine underneath. Just different directions and protocols.

```
  Notion  GitHub  Gmail  Linear  Files
     │       │       │      │      │
     └───────┴───────┴──────┴──────┘
                     │
               ← Sync (Pull)
                     │
            ╔════════▼════════╗
            ║    puppyone     ║   Context File System
            ╚════════▲════════╝
                     │
               Sync (Read / Write) →
                     │
       ┌─────────────┼─────────────┐
       │             │             │
   OpenClaw         MCP         REST API
   (↕ bidirectional) (r/w)       (r/w)
       │             │             │
  Local folder  Cursor / Claude  Custom Agent
```

MCP and REST API are not a separate "distribution layer" — they are protocol wrappers on top of the sync result. OpenClaw is the most direct form: bidirectional sync between a local folder and puppyone, kept live.

---

## File Level Security (FLS)

**Permission control down to the file path.**

Similar to Row Level Security (RLS) in databases, FLS lets you control exactly what each agent can see and do:

- Customer Support Agent: read-only `/products` and `/faq`
- Dev Agent: read/write on `/specs`, no delete
- Sales Agent: can read pricing, cannot see `/internal`

Paths an agent is not authorized to access **do not exist in its view** — physically removed from the file system it sees.
