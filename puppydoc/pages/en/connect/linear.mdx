# Linear

Sync Linear Issues and project data into structured JSON so agents can query task status, assignees, and priorities.

| | |
|---|---|
| **Direction** | Pull (Linear → PuppyOne) |
| **Trigger** | Manual |
| **Auth** | Linear OAuth |
| **Output** | JSON |

---

## Data Structure

All issues are stored in a single JSON node:

```json
{
  "issues": [
    {
      "id": "ENG-123",
      "title": "Fix login page flickering",
      "status": "in_progress",
      "priority": "urgent",
      "assignee": {
        "name": "Alice Chen",
        "email": "alice@example.com"
      },
      "labels": ["bug", "frontend"],
      "created_at": "2026-02-20T09:00:00Z",
      "updated_at": "2026-02-25T14:30:00Z",
      "due_date": "2026-03-01",
      "description": "Users report flickering on the login page in Safari...",
      "team": "Engineering",
      "cycle": "Sprint 12"
    }
  ],
  "total": 87,
  "synced_at": "2026-02-26T08:00:00Z"
}
```

---

## Setup

### Step 1: Authorize Linear

1. Open your Project → **Add Connector** → **Linear**
2. Click **Connect with Linear** and complete OAuth

### Step 2: Select Sync Scope

| Option | Description |
|--------|-------------|
| **Team** | Which team's issues to sync |
| **Status filter** | Only sync specific statuses (e.g. `in_progress`, `todo`) |
| **Time range** | Only sync recently updated issues |

### Step 3: Start Sync

Click **Start Sync**. Linear sync is manual — data is available immediately after completion.

---

## Agent Usage Example

```
User: What urgent bugs are still open?

Agent: [calls query_data, filter: priority=urgent AND status!=done]
       Found 3 urgent open bugs:
       1. ENG-123 - Login page flickering (Alice Chen, due Mar 1)
       2. ENG-156 - iOS crash (unassigned, due Feb 28)
       3. ENG-171 - Payment timeout (Bob Wang, due Mar 5)
```

---

## Notes

- Linear sync is **Pull only** — agents cannot modify Linear issues through PuppyOne
- Each sync is a full pull; larger workspaces may take a few minutes
