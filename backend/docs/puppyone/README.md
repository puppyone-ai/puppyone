## 文档规范

### 命名规则

- 使用小写字母和连字符：`folder-search.md`
- 按功能模块分目录：`turbopuffer/`、`etl/`、`mcp/` 等

### 内容结构

建议每篇文档包含以下章节：

1. **功能概述** - 背景、核心特性、架构图
2. **数据模型** - 表结构、数据格式
3. **核心实现** - 关键代码、流程说明
4. **API 接口** - 请求/响应示例
5. **测试验证** - 测试方法、验证点
6. **配置说明** - 环境变量、依赖
7. **注意事项** - 性能、成本、错误处理
8. **后续规划** - TODO 列表

## 贡献指南

2. 更新本 README 的文档索引
