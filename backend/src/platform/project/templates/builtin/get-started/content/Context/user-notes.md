# User notes

The user is evaluating PuppyOne for agent work.

They care about:

- Getting their context into a coding agent without a long setup process.
- Seeing whether the agent can write back into the same workspace.
- Knowing that the write is tracked, not lost in a local folder.
- Reusing the pattern later with their own files or GitHub repo.

The user does not want to learn every connector first. They want one clear
moment where the product becomes real.

## Good first outcome

The agent reads this repo and creates `Agent Output/first-run.md`.

That output proves the basic loop:

```text
PuppyOne context was available to the agent, and the agent wrote back.
```
